## Runtime Adaptation Fixes

### 1. Version path prefix (api.js line 1312)

The `extendAppPath()` function injects `/9.3.1-10/` into resource URLs for cache busting.
In standalone Vite deployment (no nginx rewriting), this causes all iframe resources to
404 because Vite's SPA fallback returns `text/html` for unmatched paths.

**Fix**: `const ver = '/9.3.1-10'` ‚Üí `const ver = ''`.
Must be reapplied after each api.js regeneration from template.

### 2. AllFonts.js (sdkjs/common/AllFonts.js)

The 9.3.1 `.deb` does not ship `AllFonts.js` (fonts moved to `libfont/engine/`).
The editor's require.js config still maps `"allfonts"` ‚Üí `"../../sdkjs/common/AllFonts"`.
Without this file, the SDK fails: `Script error for "allfonts"`.

**Fix**: Copied 9.3-compatible version (100 lines) from `feat/onlyoffice-9-3-runtime`.
The 7.x version (1018 lines) is incompatible with 9.3.1 font engine (`AscCommon.DN.decode`).

### 3. themes.js (sdkjs/slide/themes/themes.js)

The slide SDK loads `themes.js` for `AscCommon.g_defaultThemes`. Not shipped in .deb.

**Fix**: Created with 9.3.1-compatible theme list matching `.pptx` files at `src/`:
`['Blank','Basic','Classic','Official','Green leaf','Lines','Office','Safari','Dotted','Corner','Tort']`.
Note: "Tort" replaces "Turtle" from 7.x.
